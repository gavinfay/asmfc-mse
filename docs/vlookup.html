8.1 Summary

In previous sessions, weâve learned to do some basic wrangling and find summary information with functions in the dplyr package, which exists within the tidyverse. In this session, weâll expand our data wrangling toolkit using:

  • filter() to conditionally subset our data by rows, and
  • *_join() functions to merge data frames together
  • And weâll make a nicely formatted HTML table with kable() and kableExtra

The combination of filter() and *_join() - to return rows satisfying a condition we specify, and merging data frames by like variables - is analogous to the useful VLOOKUP function in Excel.

8.1.1 Objectives

  • Use filter() to subset data frames, returning rows that satisfy variable conditions
  • Use full_join(), left_join(), and inner_join() to merge data frames, with different endpoints in mind
  • Use filter() and *_join() as part of a wrangling sequence
  • Make an HTML table with kable() and kableExtra

8.3 dplyr::filter() to conditionally subset by rows

Use filter() to let R know which rows you want to keep or exclude, based whether or not their contents match conditions that you set for one or more variables.

Some examples in words that might inspire you to use filter():

  • âI only want to keep rows where the temperature is greater than 90Â°F.â
  • âI want to keep all observations except those where the tree type is listed as unknown.â
  • âI want to make a new subset with only data for mountain lions (the species variable) in California (the state variable).â

When we use filter(), we need to let R know a couple of things:

  • What data frame weâre filtering from
  • What condition(s) we want observations to match and/or not match in order to keep them in the new subset

Here, weâll learn some common ways to use filter().

8.3.0.1 Filter rows by matching a single character string

Letâs say we want to keep all observations from the fish data frame where the common name is âgaribaldi.â Here, we need to tell R to only keep rows from the fish data frame when the common name (common_name variable) exactly matches garibaldi.

Use == to ask R to look for exact matching strings:

fish_garibaldi <- filter(fish, common_name == "garibaldi")

Check out the fish_garibaldi object to ensure that only garibaldi observations remain.

You could also do this using the pipe operator %>% (thatâs the format weâll use moving forward in this session):

fish_garibaldi <- fish %>% 
  filter(common_name == "garibaldi")

8.3.2 Filter rows based on numeric conditions

Use expected operators (>, <, >=, <=, ==) to set conditions for a numeric variable when filtering. For this example, we only want to retain observations when the total_count column value is >= 50:

fish_over50 <- fish %>% 
  filter(total_count >= 50)

8.3.3 Filter to return rows that match this OR that OR that

What if we want to return a subset of the fish df that contains garibaldi, blacksmith OR black surfperch?

There are several ways to write an âORâ statement for filtering, which will keep any observations that match Condition A or Condition B or Condition C. In this example, we will create a subset from fish that only contains rows where the common_name is garibaldi or blacksmith or black surfperch.

Use %in% to ask R to look for any matches within a vector:

fish_3sp <- fish %>% 
  filter(common_name %in% c("garibaldi", "blacksmith", "black surfperch"))

Alternatively, you can indicate OR using the vertical line operator | to do the same thing (but you can see that itâs more repetitive when looking for matches within the same variable):

fish_3sp <- fish %>% 
  filter(common_name == "garibaldi" | common_name == "blacksmith" | common_name == "black surfperch")

8.3.4 Filter to return rows that match conditions for multiple variables

In the previous examples, we set filter conditions based on a single variable (e.g.Â common_name). What if we want to return observations that satisfy conditions for multiple variables?

For example: We want to create a subset that only returns rows from inverts where the site is âaburâ or âmohkâ and the common_name is âpurple urchin.â In filter(), add a comma (or ampersand â&â) between arguments for multiple âandâ conditions:

urchin_abur_mohk <- inverts %>% 
  filter(site %in% c("abur","mohk"), common_name == "purple urchin")
head(urchin_abur_mohk)

Like most things in R, there are other ways to do the same thing. For example, you could do the same thing using & (instead of a comma) between âandâ conditions:

# Use the ampersand (&) to add another condition "and this must be true":

urchin_abur_mohk <- inverts %>% 
  filter(site %in% c("abur","mohk") & common_name == "purple urchin")

Or you could just do two filter steps in sequence:

# Written as sequential filter steps:

urchin_abur_mohk <- inverts %>% 
  filter(site %in% c("abur", "mohk")) %>% 
  filter(common_name == "purple urchin")

8.4 Join data frames with dplyr::*_join()

Excelâs VLOOKUP can also be used to merge data from separate tables or worksheets. Here, weâll use the *_join() functions to merge separate data frames in R.

There are a number of ways to merge data frames in R. Weâll use full_join(), left_join(), and inner_join() in this session.

From R Documentation (?join):

  • full_join(): âreturns all rows and all columns from both x and y. Where there are not matching values, returns NA for the one missing.â Basically, nothing gets thrown out, even if a match doesnât exist - making full_join() the safest option for merging data frames. When in doubt, full_join().

  • left_join(): âreturn all rows from x, and all columns from x and y. Rows in x with no match in y will have NA values in the new columns. If there are multiple matches between x and y, all combinations of the matches are returned.â

  • inner_join(): âreturns all rows from x where there are matching values in y, and all columns from x and y. If there are multiple matches between x and y, all combination of the matches are returned.â This will drop observations that donât have a match between the merged data frames, which makes it a riskier merging option if youâre not sure what youâre trying to do.

Schematic (from RStudio data wrangling cheat sheet):

To clarify what the different joins are doing, letâs first make a subset of the fish data frame that only contains observations from 2016 and 2017.

fish_16_17 <- fish %>% 
  filter(year == 2016 | year == 2017)

Take a look to ensure that only those years are included with View(fish_2016_2017). Now, letâs merge it with our Arroyo Burro kelp fronds data (kelp_abur) in different ways.

8.4.1 full_join() to merge data frames, keeping everything

When we join data frames in R, we need to tell R a couple of things (and it does the hard joining work for us):

  • Which data frames we want to merge together
  • Which variables to merge by

Note: If there are exactly matching column names in the data frames youâre merging, the *_join() functions will assume that you want to join by those columns. If there are no matching column names, you can specify which columns to join by manually. Weâll do both here.

# Join the fish_16_17 and kelp_abur
abur_kelp_fish <- fish_16_17 %>% 
  full_join(kelp_abur, by = c("year", "site"))

Letâs look at the merged data frame with View(abur_kelp_fish). A few things to notice about how full_join() has worked:

  1. All columns that existed in both data frames still exist.
  2. All observations are retained, even if they donât have a match. In this case, notice that for other sites (not âaburâ) the observation for fish still exists, even though there was no corresponding kelp data to merge with it. The kelp frond data from 2018 is also returned, even though the fish counts dataset did not have âyear == 2018â in it.
  3. The kelp frond data is joined to all observations where the joining variables (year, site) are a match, which is why it is repeated 5 times for each year (once for each fish species).

Because all data (observations & columns) are retained, full_join() is the safest option if youâre unclear about how to merge data frames.

8.4.2 left_join() to merge data frames, keeping everything in the âxâ data frame and only matches from the âyâ data frame

Now, we want to keep all observations in fish_2016_2017, and merge them with kelp_abur while only keeping observations from kelp_abur that match an observation within fish_2016_2017. So when we use left_join(), any information on kelp frond counts from 2018 should be dropped, because those wouldnât have a match in the left data frame.

fish_kelp_16_17 <- fish_16_17 %>% 
  left_join(kelp_abur)
## Joining, by = c("year", "site")

Notice when you look at fish_kelp_16_17, the 2018 data that does exist in kelp_abur does not get joined to the fish_16_17 data frame, because left_join(df_a, df_b) will only keep observations from df_b if they have a match in df_a!

8.4.3 inner_join() to merge data frames, only keeping observations with a match in both

Use inner_join() if you only want to retain observations that have matches across both data frames. Caution: this is built to exclude any observations that donât match across data frames by joined variables - double check to make sure this is actually what you want to do!

For example, if we use inner_join() to merge fish and kelp_abur, then we are asking R to only return observations where the joining variables (year and site) have matches in both data frames. Letâs see what the outcome is:

kelp_fish_injoin <- fish %>% 
  inner_join(kelp_abur)
## Joining, by = c("year", "site")
# kelp_fish_injoin

Here, we see that only observations (rows) where there is a match for year and site in both data frames are returned.

8.4.4 Using filter() and join() in a sequence

Now letâs combine what weâve learned about piping, filtering and joining!

Letâs complete the following as part of a single sequence (remember, check to see what youâve produced after each step) to create a new data frame called my_fish_join:

  • Start with fish data frame
  • Filter fish to only including observations for 2017 at Arroyo Burro
  • Join the kelp_abur data frame to the resulting subset using left_join()

That sequence might look like this:

  filter(year == 2017, site == "abur") %>% 
  left_join(kelp_abur, by = c("year", "site")) %>% 
  mutate(fish_per_frond = total_count / total_fronds)

8.6 Fun facts

How is this similar to VLOOKUP in Excel? How does it differ?

From Microsoft Office Support, âuse VLOOKUP when you need to find things in a table or a range by row.â

So, both filter() and VLOOKUP look through your data frame (or spreadsheet, in Excel) to look for observations that match your conditions. But they also differ in important ways:

  1. By default VLOOKUP looks for and returns an observation for approximate matches (and you have to change the final argument to FALSE to look for an exact match). In contrast, by default filter() will look for exact conditional matches.

  2. VLOOKUP will look for and return information from the first observation that matches (or approximately matches) a condition. filter() will return all observations (rows) that exactly match a condition.
